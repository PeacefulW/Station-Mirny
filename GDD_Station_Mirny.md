# Станция «Мирный» — Game Design Document v1.1

> **Статус:** Препродакшн — живой документ, обновляется по мере разработки
> **Движок:** Godot 4.x (GDScript)
> **Жанр:** 2D Top-Down Survival / Base-Builder с элементами автоматизации и RPG
> **Платформа:** PC (Steam) — основная. Кооп на 2–4 игрока в будущем
> **Целевое время прохождения:** 100+ часов (одна ветка технологий)

---

## 1. Видение проекта

### 1.1 Core Fantasy

**«В базе хорошо, снаружи страшно.»**

Игрок — инженер, единственный выживший после крушения исследовательского модуля на враждебной планете. Воздух отравлен спорами, кислорода критически мало, ночью температура падает до смертельных значений. Единственный шанс — построить герметичную базу, наладить жизнеобеспечение и шаг за шагом расширять островок безопасности посреди враждебного мира.

Каждый выход за стены — это напряжение. Каждое возвращение — облегчение. База — это не просто набор построек, это *дом*, который игрок создал своими руками и защищает зубами.

### 1.2 Уникальные продающие точки (USP)

1. **Два пути развития с разным геймплеем и эстетикой базы:**
   - Путь Терраформера (Техногенный) — индустриальный бункер, тяжёлая техника, подчинение планеты силой
   - Путь Адаптации (Биологический) — симбиоз с природой, мутации инженера, приручение фауны
2. **Лоровая система сохранений** — игрок открывает возможность «сохраняться» через игровую механику (кровь местных существ), а не через меню
3. **Дешифровка вместо исследований** — технологии не изучаются с нуля, а восстанавливаются из повреждённых архивов данных Ковчега, что требует вычислительных мощностей
4. **Живая враждебная среда** — споры засоряют фильтры, проникают через повреждённые стены, вызывают рост биомассы внутри базы
5. **Глубокий лор с главным твистом** — планета не чужая, она *наша*, и споры пытаются «вернуть» человека к изначальному виду

### 1.3 Референсы по ощущениям

| Аспект | Референс |
|--------|----------|
| Атмосфера | «Сталкер» × «Интерстеллар» — мрачная красота враждебного мира |
| Строительство базы | RimWorld (тайловые стены) + Oxygen Not Included (трубы, газы) |
| Управление | Factorio (прямое управление одним персонажем) |
| Автоматизация | Factorio-лайт: автоматизация до сырья, крафт вручную |
| Выживание | Don't Starve (голод, холод) + Subnautica (исследование, лор) |
| Эвенты | RimWorld (случайные события, нарастающая сложность) |
| Вылазки / данжи | Risk of Rain 2 (напряжение) + Subnautica (разведка, лор через окружение) |

## 2. Лор и сеттинг

### 2.1 Предыстория

Земля погибла. Человечество выжило на орбитальном Ковчеге — гигантском корабле-городе, несущем последние остатки цивилизации. Чтобы найти новый дом, Ковчег рассылает во все концы космоса подготовленных специалистов — **Инженеров**. Каждый — самостоятельная единица: учёный, строитель, солдат в одном лице. Каждый несёт с собой блок данных с полной базой знаний человечества, оборудование для базового лагеря и исследовательскую аппаратуру.

Наш инженер — один из многих. Его модуль вошёл в атмосферу планеты-кандидата, но что-то пошло не так. Жёсткая посадка. Обломки раскидало по округе. Часть оборудования уцелела. Блок данных повреждён — терабайты чертежей и формул превратились в зашифрованный мусор. Связь с Ковчегом потеряна. Обратного пути нет.

Единственный выбор — выжить. И, возможно, сделать эту планету домом.

### 2.2 Главная тайна (СПОЙЛЕР — закрытая информация)

> **Раскрывается постепенно через 60–80% прохождения.**

Планета — не чужая. Это **прародина человечества**. Древняя цивилизация людей жила здесь задолго до Земли. Споры — это вышедшая из-под контроля древняя система биологического терраформирования (или оружие), которая когда-то уничтожила ту цивилизацию и заставила предков бежать — сначала на Луну, потом на Землю.

Споры не пытаются убить инженера. Они пытаются **«откатить» его ДНК** к изначальному виду — к тому, чем были люди до бегства.

**Цепочка раскрытия:**
1. Ранняя игра — странные барельефы в руинах, анатомия «чужих» подозрительно похожа на человеческую
2. Мидгейм — сканер базы анализирует местную флору и выдаёт ошибку: *«Совпадение генома с объектом [Человек_Инженер] — 99.8%»*
3. Поздняя игра — полная расшифровка архивов Предтеч подтверждает: мы вернулись домой

**Влияние на два пути:**
- **Терраформер** — осознанно уничтожает наследие предков ради выживания по-своему. Трагический героизм
- **Адаптация** — принимает наследие, трансформируется, становится тем, кем человечество было изначально. Возвращение к истокам

### 2.3 Планета

**Визуальный стиль:** Оранжевые тона, высокая колышущаяся трава, постоянный ветер, пасмурное небо с редкими лучами солнца. Мрачная осенняя красота.

**Физические условия:**
- Атмосфера: ~5% кислорода (земная норма — 21%). Без маски — гипоксия за минуты
- Споры: повсеместны, как мелкая пыль. Токсичны при вдыхании, засоряют механизмы
- Температура: холодная «весна» как базовая. Ночью — значительное падение. Сезонные колебания
- Гравитация: близка к земной (возможны отличия по биомам — TBD)

**Биомы (процедурная генерация):**
- Оранжевые равнины (стартовый) — высокая трава, умеренные споры, базовые ресурсы
- Споровые леса — грибовидные деревья, высокая концентрация спор, ценная биомасса
- Ледяные пустоши — мало спор, но смертельный холод. Редкие минералы
- Вулканические зоны — геотермальная энергия, опасные газы, уникальные руды
- Руины Предтеч — поверхностные структуры, входы в подземные комплексы
- *(Дополнительные биомы через моды и обновления)*

## 3. Основной игровой цикл

### 3.1 Петля геймплея

```
Утро: Проверка базы → Починка фильтров → Планирование вылазки
       ↓
День: Выход в скафандре → Сбор ресурсов / Разведка / Данж
       ↓
Вечер: Возвращение → Шлюзование → Разгрузка → Крафт на верстаке
       ↓
Ночь: База под осадой? → Оборона. Тихая ночь? → Дешифровка технологий, улучшение базы
       ↓
(Цикл повторяется с нарастающей сложностью и масштабом)
```

### 3.2 Прогрессия по этапам

**Ранняя игра (0–10 часов):**
- Сбор обломков корабля вокруг места крушения
- Постройка первого укрытия: 4 стены, крыша, примитивный шлюз
- Ручная добыча ресурсов, кузница для базовых слитков
- Бластер разряжается — переход на примитивное оружие
- Первое столкновение с фауной
- Обнаружение существ с «особой кровью» → выбор: попробовать или нет (система сохранений)

**Мидгейм (10–50 часов):**
- Компрессоры, фильтры, система труб — автоматизация жизнеобеспечения
- Автоматическая добыча руды, выплавка слитков
- Первый транспорт (вездеход)
- Дешифровка продвинутых технологий (нужно много электричества)
- Развилка путей: Терраформер или Адаптация
- Исследование данжей Предтеч
- Постройка аванпостов, рельсовые соединения

**Эндгейм (50–100+ часов):**
- **Путь Терраформера:** Гигантские атмосферные процессоры, щитовые генераторы, тяжёлая бронетехника. Финал — запуск глобального терраформирования
- **Путь Адаптации:** Мутации инженера, приручённые существа-транспорт, биолаборатории. Финал — полная адаптация, инженер снимает шлем
- Раскрытие главной тайны
- Возможность перелёта на другую планету (новый seed, новые правила генерации)
- Песочница: продолжение строительства и оптимизации после финала

### 3.3 Цикл дня и ночи

- **День:** Относительно безопасно для вылазок (в пределах возможностей скафандра). Фауна менее активна. Солнечные панели работают
- **Ночь:** Температура падает. Больше агрессивных существ. Очистители активизируются. Повышенная концентрация спор. Рискованно выходить без подготовки
- **Длительность цикла:** ~30 минут реального времени (настраиваемо)

### 3.4 Сезоны

Полный цикл из 4 сезонов, каждый влияет на геймплей:

- **Тёплый сезон (аналог лета):** Максимальная солнечная генерация, слабый ветер, низкая активность фауны. Лучшее время для экспедиций
- **Сезон спор (аналог осени):** Пиковое спорообразование, фильтры засоряются быстрее, видимость снижена. Богатый сбор биоматериалов
- **Холодный сезон (аналог зимы):** Экстремальные температуры, повышенный расход энергии на обогрев. Споры менее активны. Некоторые существа впадают в спячку
- **Сезон штормов (аналог весны):** Сильный ветер (ветряки на максимуме), грозы, землетрясения. Возможны повреждения наружных конструкций

## 4. Выживание

### 4.1 Основные шкалы

| Параметр | Описание | Последствия при критическом уровне |
|----------|----------|------------------------------------|
| **Кислород (O₂)** | Запас в баллоне скафандра | Гипоксия: падение выносливости, размытие экрана, замедление, потеря сознания, смерть |
| **Токсичность** | Накопление спор в организме | Кашель (демаскирует), галлюцинации (тени), болезни, смерть. Лечится антидотами |
| **Температура тела** | Зависит от погоды, одежды, обогрева | Обморожение: замедление, потеря HP. Перегрев: потеря жидкости |
| **Голод** | Расходуется постоянно, быстрее при нагрузке | Потеря выносливости, замедление регенерации, смерть |
| **Жажда** | Вода нужна чистая — загрязнённая повышает токсичность | Как голод, но быстрее |
| **Усталость** | Растёт от действий, падает от сна | Снижение скорости, точности, риск потери сознания |

### 4.2 Воздух и споры

**Механика гипоксии:**
Кислород есть, но его мало (~5%). Без маски или с пустым баллоном персонаж не умирает мгновенно. Сначала резко падает выносливость, экран размывается по краям, движения становятся вязкими. Создаётся напряжение: ушёл слишком далеко, кислород на исходе, пытаешься бежать к базе, но гипоксия позволяет только медленно плестись.

**Споры — двойная угроза:**

1. *Угроза инженеру (интоксикация):* Вдыхание повышает шкалу Токсичности. Эффекты: кашель (привлекает хищников), лёгкие галлюцинации (тени по краям экрана), болезни. Лечение — синтезированные антидоты на базе

2. *Угроза механизмам (засорение):* Компрессоры засасывают атмосферный воздух для отделения O₂. Споры забивают фильтры. Грязевики и фильтры грубой очистки нужно регулярно менять или чистить. Забыл — воздух перестал поступать в жилой отсек

**Шлюзовая система:**
Вход в базу — полноценный шлюз:
1. Вошёл в первую камеру → дверь закрылась
2. Система откачала грязный воздух
3. Обдув дезинфицирующим газом / прожиг ультрафиолетом (убивает споры на скафандре)
4. Открывается внутренняя дверь

**Прорыв периметра:**
Если стена повреждена или шлюз сломался — споры попадают внутрь. По стенам жилого отсека начинает ползти чужеродная биомасса. Она замыкает проводку, портит гидропонику, отравляет воздух. Игрок надевает маску *внутри собственного дома* и вычищает заразу огнемётом или химикатами.

### 4.3 Еда и вода

**Источники пищи:**
- Гидропоника / теплицы на базе — основной надёжный источник. Требует воды, света, тепла
- Охота на местную фауну — рискованно, но питательно. Мясо нужно проверять на совместимость
- Собирательство местных растений — некоторые съедобны после обработки, другие ядовиты
- Синтетическая еда — из переработки биомассы на продвинутом оборудовании

**Вода:**
- Добывается из подземных источников или конденсацией из атмосферы
- Сырая вода загрязнена (споры, минералы) — нужна очистка
- Система фильтрации связана с трубопроводом

### 4.4 Система смерти и сохранений

**Ранняя игра (до обнаружения «особой крови»):**
Зависит от выбранного режима сложности:
- **Инженер (хардкор):** Чистый permadeath. Смерть = новая игра
- **Колонист (стандарт):** Сейв при выходе из игры (save on quit)
- **Турист (лёгкий):** Автосейв каждые игровые сутки

**После обнаружения крови (лоровая механика):**
Убив определённое существо, анализатор обнаруживает «странные элементы» в его крови. Выбор: попробовать или нет.

Если выпить — эффект ноль. Но при смерти инженер «откатывается» к моменту, когда выпил кровь. Механически — это лоровый чекпоинт.

**Ограничения:** Повторное употребление повышает токсичность. Нельзя пить бесконечно.

**Дальнейшие исследования (мидгейм):**
Лаборатория позволяет синтезировать зелье на основе крови — без токсичности. Выпив зелье, игрок создаёт точку сохранения в любом месте. Механически — ручной сейв через лоровый предмет.

## 5. Мир и генерация

### 5.1 Процедурная генерация

**Технология:** FastNoiseLite (встроен в Godot). Многослойный шум для разных параметров.

**Слои генерации:**
1. Карта высот → рельеф (равнины, холмы, горы, впадины)
2. Карта температуры → климатические зоны
3. Карта влажности → тип растительности
4. Карта концентрации спор → плотность споровых облаков
5. Пересечение параметров → биомы

**Чанковая система:**
- Мир делится на чанки 64×64 тайла
- Загружены только чанки вокруг игрока (радиус 3–5 чанков)
- Далёкие чанки выгружаются, но seed сохраняется → повторная генерация идентична
- Изменения игрока (постройки, добыча) сохраняются в файл отдельно от seed

**Масштаб:** Технически «бесконечная» карта (ограничена только размером seed). Практически — игрок исследует столько, сколько захочет.

### 5.2 Другие планеты

Инженер может построить корабль и улететь на другую планету. Технически:
- Новая планета = новый seed + изменённые правила генерации (другие биомы, другая палитра, другие условия)
- Данные о предыдущей планете сохраняются на диск
- Возврат возможен — база на месте

### 5.3 Точки интереса (POI)

Генерируются процедурно с правилами размещения:

| POI | Где появляется | Что содержит |
|-----|----------------|--------------|
| Обломки корабля | Вокруг точки старта (ближний радиус) | Начальные ресурсы: провода, микросхемы, запчасти |
| Залежи руды | Все биомы, чаще в горах | Сырьё для выплавки |
| Подземные источники | Низины, впадины | Вода (загрязнённая) |
| Руины Предтеч (поверхность) | Редко, все биомы | Лор (барельефы, записи), редкие материалы |
| Комплексы Предтеч (данжи) | Под руинами, редко | Чертежи супертехнологий, лор, опасность |
| Гнёзда фауны | Споровые леса, пещеры | Биоматериалы, опасность |
| Геотермальные источники | Вулканические зоны | Энергия, редкие минералы |

## 6. Фауна

### 6.1 Концепция

Животные планеты — не просто звери. Это **одичавшие биоинженерные инструменты** древней цивилизации. Каждый вид когда-то выполнял функцию в экосистеме Предтеч. Теперь они дикие, но их «программа» всё ещё работает.

### 6.2 Касты существ

**Очистители (агрессивные):**
- Слепые твари с гипертрофированным слухом
- Изначальная функция: санитары, уничтожающие биомусор
- Сейчас: «мусор» — это инженер и шум его генераторов
- Реагируют на вибрацию механизмов → атакуют базу
- Активнее ночью
- Геймплейная роль: основная боевая угроза, мотивация строить тихие механизмы или шумоподавление

**Фильтровальщики (мирные / полезные):**
- Огромные, медлительные существа — помесь черепахи и грибницы
- Изначальная функция: биологические очистители атмосферы
- Сейчас: вдыхают споры, выдыхают чистый кислород
- Игнорируют инженера
- Геймплейная роль: маркер чистого воздуха. На пути Адаптации — потенциальный транспорт

**Другие виды (TBD — расширяется через обновления и моды):**
- Рытели — подземные существа, создающие тоннели. Опасны для фундамента базы
- Светляки — мелкие, мирные, светятся. Индикатор безопасных зон
- Голиафы — гигантские биомеханические машины, размером с гору. Не замечают инженера, но меняют ландшафт

### 6.3 Взаимодействие по путям

| Действие | Путь Терраформера | Путь Адаптации |
|----------|-------------------|----------------|
| Очистители | Уничтожать, перерабатывать на биотопливо | Изучать, создавать репелленты |
| Фильтровальщики | Игнорировать или перерабатывать | Приручать: транспорт с аурой чистого воздуха |
| Общий подход | Техника: вездеходы с бурами и пулемётами | Биология: сёдла, системы управления, симбиоз |

## 7. Строительство базы

### 7.1 Система строительства

**Тип:** Тайловое строительство в стиле RimWorld. Стены, полы, двери, предметы ставятся по сетке свободно.

**Многоэтажность:**
- Можно строить вниз (бункер) — до нескольких подземных уровней
- Подземные уровни безопаснее: хищники не доберутся, стабильная температура
- Можно строить вверх (ограниченно — 2–3 этажа) для обзорных вышек, антенн
- Данжи Предтеч — отдельные подземные уровни, не строятся игроком

**Герметичность:**
- Замкнутые стены + крыша = герметичное помещение
- Внутри можно создать пригодную атмосферу
- Любая дыра (повреждение, отсутствие стены) = разгерметизация
- Шлюз обязателен для входа/выхода

### 7.2 Масштаб

**Эндгейм-база:** Компактный бункер на 10–15 функциональных помещений. Каждая комната — на счету. Акцент на эффективности и уюте, не на масштабе.

**Типичные помещения:**
- Шлюз (обязательно)
- Жилой отсек (кровать, хранение личных вещей)
- Мастерская (верстаки, сборка)
- Компрессорная (воздухоочистка)
- Водоочистная
- Электрощитовая / серверная (дешифровка данных)
- Теплица / гидропоника
- Лаборатория (исследования, синтез зелий)
- Склад
- Арсенал
- Медотсек
- *(На пути Адаптации: Биолаборатория, Инкубатор)*
- *(На пути Терраформера: Тяжёлый цех, Генераторная)*

### 7.3 Аванпосты

Помимо основной базы, игрок может строить аванпосты:
- Упрощённые конструкции (укрытие + минимальное жизнеобеспечение)
- Добывающие станции на удалённых месторождениях
- Соединяются рельсовой системой с основной базой
- Вагонетки автоматически перевозят ресурсы между аванпостами

### 7.4 Инженерные системы (трубы и провода)

**Три типа коммуникаций:**
1. **Воздуховоды** — подача очищенного воздуха от компрессоров в помещения
2. **Водопровод** — подача чистой воды к теплицам, медотсеку, жилым комнатам
3. **Электропроводка** — подача энергии от генераторов к потребителям

**Отображение:**
- Отдельный режим просмотра (overlay) для прокладки коммуникаций
- В обычном режиме трубы и провода скрыты под текстурами
- Прокладываются по стенам (обязательное условие)
- При повреждении стены — повреждаются коммуникации внутри

### 7.5 Электроснабжение

**Философия:** Источники энергии отражают мир, а не Землю. Никаких солнечных панелей и ветряков — на этой планете свои правила. Каждый источник рассказывает историю о мире и создаёт уникальные геймплейные решения.

**Источники энергии (от ранних к поздним):**

| Источник | Доступность | Описание | Плюсы | Минусы |
|----------|-------------|----------|-------|--------|
| Аварийные батареи Ковчега | Стартовая | Уцелевшие энергоячейки из обломков корабля. Конечный ресурс — когда кончатся, кончатся навсегда. Знакомый якорь: «наша технология, она понятна» | Стабильные, бесшумные, надёжные | Конечный запас, невозможно восстановить |
| Термосжигатель | Ранняя | Примитивная печь, сжигающая местную биомассу (древесину, сухую траву). Единственный «земной» генератор в игре — дальше всё будет чужим | Из подручных материалов, крафтится рано | Шумный (привлекает Очистителей), нужно топливо |
| Споровый ферментатор | Средняя | Споры — главная угроза, но при ферментации выделяют энергию. То, что тебя убивает — теперь питает базу. Красивая ирония | Топливо повсюду (споры), тихий | Нужны сменные фильтры, медленная генерация |
| Сейсмоуловитель | Средняя | Пьезоэлектрический преобразователь. Планета постоянно дрожит — Голиафы бродят, тектоника активна. Вибрация грунта → электричество | Бесплатная энергия, бесшумный | Нестабильный выход, зависит от сейсмоактивности зоны |
| Электростатический коллектор | Средняя | Антенная мачта, собирающая статический заряд атмосферы, насыщенной спорами. Пиковая мощность в Сезон штормов | Огромная мощность в шторм | Зависит от погоды, заметный (привлекает внимание) |
| Кондуит Предтеч | Поздняя | Рабочие энергоузлы древней цивилизации в руинах. Подключение к сети, которая работает тысячелетия. Приближает к главной тайне | Огромная мощность, лор-значимость | Только у руин, непредсказуемые скачки напряжения |
| Геотермальный кран | Поздняя | Бурение к жару недр в вулканических зонах | Стабильная и мощная | Привязка к вулканической локации |
| *(Путь Терраформера: Плазменный реактор)* | Эндгейм | Венец человеческой инженерии — удержание сверхнагретой плазмы в магнитной ловушке. Чертежи дешифруются из архивов Ковчега | Колоссальная мощность | Сложнейший крафт, при аварии — катастрофа |
| *(Путь Адаптации: Корень-сердце)* | Эндгейм | Живой организм-симбионт, вросший в фундамент базы. Питается спорами, отдаёт биоэлектричество. Пульсирует, растёт, чинит себя сам | Самовосстановление, питается спорами | Нужен живой уход, медленное развёртывание |

**Геймплейные дилеммы энергосистемы:**
- Споровый ферментатор бесшумный, но нужны фильтры. Термосжигатель не нужен, но шумит → Очистители
- Электростатический коллектор даёт пик в шторм, но шторм повреждает базу → риск vs. награда
- Кондуит Предтеч — самый мощный «бесплатный» источник, но находится в опасной зоне → нужен аванпост
- Корень-сердце (Адаптация) питается спорами → чем хуже снаружи, тем больше энергии. База буквально питается враждебностью мира

**Энергобаланс:** Ключевая механика. Дешифровка технологий требует серверов, серверы потребляют огромное количество энергии. Прогрессия по дереву технологий напрямую привязана к энергосистеме базы.

## 8. Автоматизация

### 8.1 Философия

**«Автоматизация до сырья, крафт — руками.»**

Автоматизируется рутина добычи и переработки. Финальная сборка — всегда на верстаке лично. Это не завод, а мастерская инженера.

### 8.2 Что автоматизируется

| Процесс | Как |
|----------|-----|
| Добыча руды | Электрический бур на месторождении |
| Добыча воды | Насос на подземном источнике |
| Фильтрация воздуха | Компрессор + фильтры (требуют замены) |
| Очистка воды | Фильтрационная установка |
| Выплавка слитков | Электрическая печь (руда → слиток автоматически) |
| Транспортировка между базами | Рельсы + вагонетки |
| Обогрев / охлаждение | Системы климат-контроля |

### 8.3 Что делается вручную

- Крафт предметов на верстаке (патроны, детали, оружие, мебель, модули скафандра)
- Сборка сложных устройств
- Замена фильтров (пока не автоматизируешь через роботов — TBD)
- Ремонт повреждений
- Исследование / дешифровка (запуск процесса вручную)

### 8.4 Рельсовая система

- Строятся рельсы между базой и аванпостами
- Вагонетки перевозят ресурсы автоматически по расписанию или при заполнении
- Можно настраивать маршруты и приоритеты
- Рельсы нужно защищать (фауна может повредить)

## 9. Защита базы

### 9.1 Философия

**Комбинация: автоматическая защита + личное участие.**

Турели и ловушки держат мелкие угрозы, но серьёзные атаки требуют, чтобы игрок взял оружие и вышел на стены.

### 9.2 Оборонительные структуры

**Пассивная защита:**
- Стены (разные материалы, разная прочность)
- Укреплённые стены и двери
- Шлюзовая система (замедляет проникновение)
- Рвы / траншеи (замедляют наземную фауну)

**Активная защита:**
- Автоматические турели (требуют энергии и боеприпасов)
- Шумовые приманки (отвлекают Очистителей от базы)
- Электрозаграждения (потребляют много энергии)
- *(Путь Терраформера: Огнемётные турели, плазменные барьеры)*
- *(Путь Адаптации: Феромонные отпугиватели, симбиотические стражи)*

**Личное участие:**
- Игрок может стрелять со стен / из бойниц
- Ручное управление тяжёлой турелью
- Выход за периметр для контратаки
- Ремонт повреждений во время боя

### 9.3 Шум как механика

Генераторы, буры, конвейеры создают вибрацию. Очистители реагируют на шум. Это создаёт дилемму: больше автоматизации = больше шума = больше атак. Игрок должен балансировать между эффективностью и скрытностью, или инвестировать в шумоподавление.

## 10. Персонаж и прогрессия

### 10.1 Базовые характеристики (статы)

| Стат | Описание | На что влияет |
|------|----------|---------------|
| **Сила** | Физическая мощь | Грузоподъёмность, урон в ближнем бою, скорость строительства |
| **Выносливость** | Запас жизненных сил | Очки здоровья, выносливость (стамина), базовое сопротивление холоду |
| **Интеллект** | Умственные способности | Скорость дешифровки, качество крафта, эффективность анализа |
| **Устойчивость к спорам** | Биологическая резистентность | Скорость накопления токсичности, эффективность антидотов |
| **Эффективность O₂** | Адаптация лёгких | Длительность баллона кислорода, скорость восстановления при гипоксии |
| **Терморегуляция** | Адаптация к температуре | Расход энергии скафандра на обогрев, сопротивление холоду/жаре |

**Прокачка:** «Что делаешь — то и развивается» (система Skyrim). Таскаешь тяжести — растёт Сила. Ходишь на мороз — растёт Терморегуляция. Долго контактируешь со спорами — повышается Устойчивость. Дешифруешь данные — растёт Интеллект.

### 10.2 Навыки (скиллы)

Система use-based: действие → опыт в соответствующем навыке → повышение уровня навыка → разблокировка бонусов.

**Древо навыков (примерное):**
- **Инженерия:** Строительство, ремонт, улучшение построек
- **Металлургия:** Выплавка, обработка металлов, качество деталей
- **Электроника:** Работа с проводкой, эффективность приборов
- **Оружейное дело:** Крафт и модификация оружия, точность
- **Биология:** Анализ флоры/фауны, синтез зелий, медицина
- **Выживание:** Эффективность пищи, скорость в скафандре, ориентирование
- *(Путь Адаптации: Мутагенез, Приручение, Симбиотика)*
- *(Путь Терраформера: Тяжёлая техника, Автоматизация, Энергетика)*

### 10.3 Экипировка

**Скафандр (модульная система):**
Базовый скафандр с модульными слотами:
- Слот фильтра — тип и качество фильтрации спор
- Слот баллона O₂ — объём запаса кислорода
- Слот батареи — запас энергии (обогрев, фонарь, HUD)
- Слот брони — защита от урона
- Слот утилиты — дополнительный модуль (ночное видение, сканер, доп. карманы)

**Оружие:**
- *Стартовое:* Энергетический бластер (мощный, но заряд кончается, и без станции зарядки — бесполезен)
- *Раннее:* Примитивное (копьё, лук из местных материалов)
- *Среднее:* Огнестрельное (пистолет, винтовка — крафтятся, нужны патроны)
- *Позднее:* Зависит от пути
  - Терраформер: Тяжёлое оружие, плазменное, огнемёт
  - Адаптация: Биологическое оружие (кислотные снаряды, феромонные гранаты, симбиотические щиты)

### 10.4 Инвентарь

**Система: Слоты + ограничение по весу**
- Фиксированное количество слотов (рюкзак), расширяемое через улучшения
- Каждый предмет имеет вес
- Превышение лимита веса → замедление, повышенный расход O₂ и стамины
- На базе — хранилище (сундуки, стеллажи)

### 10.5 Трансформация на пути Адаптации

**Механика:** Через лабораторию синтезируются сыворотки на основе генома местной фауны. Каждая сыворотка необратимо изменяет инженера.

**Визуальные изменения:** Персонаж постепенно трансформируется:
- Ранние мутации: усиление мышечной массы, изменение цвета кожи
- Средние: рост увеличивается, появляются новые черты (адаптированные лёгкие — видно на модели)
- Поздние: инженер становится «Изначальным» — тем, чем люди были до бегства с планеты. Выше, мощнее, другие черты лица. Визуально — «Бог» (дизайн TBD)

**Способности (разблокируются поэтапно):**
- Повышенная устойчивость к спорам → снижение расхода фильтров
- Адаптированные лёгкие → дыхание без маски в зонах с умеренной концентрацией спор
- Ночное зрение → исследование без фонаря
- Полная адаптация (эндгейм) → шлем не нужен, свободное перемещение по планете

## 11. Крафт и технологии

### 11.1 Система крафта

**Реалистичная цепочка: Руда → Слиток → Деталь → Сборка**

Пример: создание винтовки
```
Железная руда → [Печь] → Железный слиток
Железный слиток → [Верстак] → Ствол, Затворная рама, Пружины
Медная руда → [Печь] → Медный слиток → [Верстак] → Гильзы
Ствол + Рама + Пружины + Приклад (дерево) → [Верстак сборки] → Винтовка
```

**Рабочие станции (от ранних к поздним):**
1. Примитивный верстак — базовые инструменты, камень/дерево
2. Кузница — ручная выплавка (ранняя игра, до электричества)
3. Электрическая печь — автоматическая выплавка
4. Механический верстак — детали из слитков
5. Верстак сборки — финальные предметы
6. Электронный стенд — схемы, чипы, платы
7. Химическая станция — зелья, антидоты, топливо
8. *(Адаптация: Биолаборатория — сыворотки, феромоны)*
9. *(Терраформер: Инженерный стенд — тяжёлая техника)*

### 11.2 Дерево технологий (Дешифровка)

**Лоровое обоснование:** Инженер прилетел с Ковчега, где знания человечества хранились в блоке данных. При крушении блок повреждён. Технологии не изучаются с нуля — они **дешифруются из повреждённых архивов**.

**Механика:**
- Серверная на базе — основной инструмент дешифровки
- Каждая технология требует: вычислительных циклов (время) + электроэнергии
- Чем сложнее технология — тем больше энергии и времени
- Можно добавлять серверные стойки для ускорения (но больше потребление)
- Данные из данжей Предтеч ускоряют дешифровку определённых веток

**Структура дерева:**
```
                    [Базовое выживание]
                     /              \
            [Электроника]      [Материалы]
               /     \           /      \
     [Серверная] [Оборудование] [Металлургия] [Химия]
          |           |             |           |
     [Дешифровка]  [Оружие]    [Строительство] [Медицина]
          |           |             |           |
   ┌──────┴──────┐    |        [Транспорт]  [Биология]
   |             |    |             |           |
[ТЕРРАФОРМЕР] [АДАПТАЦИЯ]     [Рельсы]    [Сыворотки]
   |             |                          |
[Атмосферные  [Мутагенез]              [Трансформация]
 процессоры]
```

Дерево глубокое (100+ часов). Ветки Терраформера и Адаптации расходятся примерно в мидгейме. Некоторые технологии можно изучить на обоих путях, но финальные — эксклюзивны.

## 12. Транспорт

| Транспорт | Этап | Описание |
|-----------|------|----------|
| Пешком | Всегда | Базовое перемещение. Медленно, уязвимо |
| Экзоскелет | Ранняя-средняя | Надевается поверх скафандра. Быстрее бег, больше грузоподъёмность. Расходует энергию |
| Вездеход / багги | Средняя | Строится на базе. Быстрый, вместительный. Нужно топливо. Шумный |
| Рельсовая вагонетка | Средняя-поздняя | Между базой и аванпостами. Автоматическая доставка ресурсов |
| *(Адаптация: Верховой Фильтровальщик)* | Средняя-поздняя | Приручённое существо. Аура чистого воздуха. Медлительный, но безопасный |
| *(Терраформер: Бронетранспортёр)* | Поздняя | Тяжёлый, вооружённый, герметичный. Топливо |
| Корабль (межпланетный) | Эндгейм | Перелёт на другую планету |

## 13. События (эвенты)

### 13.1 Философия

Эвенты — не просто «случилась буря». Каждое событие связано с лором планеты и создаёт одновременно угрозу и возможность.

### 13.2 Список событий

**«Эхо Предтеч»**
- Триггер: Геомагнитная активность
- Что происходит: Старые ретрансляторы в коре планеты просыпаются. Радио в шлеме и на базе транслирует жуткие искажённые обрывки фраз на пра-языке — крики о помощи, приказы об эвакуации
- Геймплей: На карте появляются временные зоны с редчайшим лутом. Ограниченное время
- Атмосфера: Тревога, жуть, интрига

**«Синхронизация спор»**
- Триггер: Раз в несколько игровых недель
- Что происходит: Споры светятся в темноте голубым светом. Невероятно красиво
- Геймплей: Мутанты становятся вдвое быстрее и агрессивнее, стягиваются к источникам тепла (к базе). Одновременно — споры в эту ночь менее токсичны (можно собрать ценные образцы)
- Атмосфера: Красота + ужас. «Самая красивая ночь — самая опасная»

**«Мёртвый Голиаф»**
- Триггер: Редко (раз в сезон?)
- Что происходит: Земля трясётся. На горизонте медленно проходит гигантская биомеханическая машина терраформеров размером с гору
- Геймплей: Не замечает инженера, но оставляет кратеры с ценными ресурсами, обнажает входы в новые данжи, полностью меняет ландшафт сектора
- Атмосфера: Благоговение, масштаб, ничтожность человека

**Другие эвенты (TBD):**
- Магнитная буря — отключение электроники на время
- Сезонная миграция — массовое перемещение фауны через территорию базы
- Метеоритный дождь — повреждения поверхностных построек, но редкие ресурсы в метеоритах
- Прорыв спор — резкий скачок концентрации спор, перегрузка фильтров
- Сигнал от другого инженера (лоровый квест — TBD)

## 14. Данжи — Комплексы Предтеч

### 14.1 Структура

Подземные бункеры древней цивилизации. Многоуровневые, процедурно генерируемые (при сохранении ключевых лоровых комнат).

### 14.2 Геймплей в данже

1. **Вход:** Обнаружение на поверхности → спуск
2. **Темнота:** Внутри нет спор, но нет и энергии. Жуткая темнота, только фонарь
3. **Подключение:** Находишь энергоузел → подключаешь свой портативный генератор к древней сети
4. **Свет:** Загорается освещение, активируются древние консоли с лором и чертежами
5. **Пробуждение:** Вместе со светом просыпаются защитные системы или древние био-мутанты, запертые тысячелетиями
6. **Выбор:** Забрать данные и сбежать? Или попытаться зачистить и исследовать полностью?

### 14.3 Награды

- Чертежи супертехнологий (ускоряют дешифровку)
- Лор-записи (раскрытие истории)
- Редкие материалы Предтеч
- Технологии, недоступные через обычную дешифровку

## 15. Интерфейс (UI/UX)

### 15.1 HUD

**Стиль: Минимальный HUD + PDA-наручник**

**На экране (всегда):**
- Шкала O₂ (на шлеме скафандра — диегетично)
- Индикатор заряда батареи скафандра
- Мини-компас / маркер базы
- Иконка текущего состояния (холод, токсичность — только при угрозе)

**На наручнике (PDA, открывается по кнопке):**
- Карта (открывается по мере исследования)
- Инвентарь
- Статы персонажа и навыки
- Дерево дешифровки
- Журнал (лор-записи, квесты)
- Настройки базы (производство, логистика)

**Строительный интерфейс:**
- Отдельный режим (кнопка переключения)
- Призрачная сетка для размещения
- Overlay-режим для труб/проводов/воздуховодов

### 15.2 Графический стиль

**Реалистичный 2D (painted sprites)**
- Рисованные спрайты с детализированными текстурами
- Цветовая палитра: приглушённые оранжевые, серые, тёмно-зелёные тона снаружи. Тёплые искусственные тона внутри базы
- Контраст: база (тёплое освещение, чистые поверхности) vs. внешний мир (холодный, мрачный, туманный)
- Эффекты шейдеров: ветер на траве/кустах, туман спор, мерцание освещения
- Система частиц: пар изо рта, искры, дым, споровые облака

### 15.3 Арт-пайплайн

**Философия:** Движок — 2D (Godot 4), но визуальный стиль — «объёмный 2D». Спрайты выглядят как 3D-объекты благодаря pre-rendered подходу (как в Factorio, Diablo 2, RimWorld). Это даёт лучшее из обоих миров: объёмный вид при простоте и производительности 2D-движка.

**Два типа графических ассетов:**

**1. Объекты (персонажи, фауна, техника, мебель) — 3D → 2D пайплайн:**

```
Gemini (4 ракурса: перед, зад, лево, право)
    ↓
Tripo3D (генерация 3D-модели из 4 картинок)
    ↓
Blender (скрипты для рендера):
  → Спрайтшиты: все направления поворота (8–16 углов)
  → Анимации: ходьба, атака, крафт, смерть
  → Normal maps: для 2D-освещения в Godot
  → Тени: запечённые тени для статичных объектов
    ↓
Godot (Sprite2D + AnimationPlayer)
```

**Что рендерится через 3D-пайплайн:**
- Инженер (все анимации, все направления, трансформация на пути Адаптации)
- Очистители, Фильтровальщики, Голиафы и прочая фауна
- Вездеход, экзоскелет, вагонетки
- Мебель и оборудование внутри базы (верстаки, печи, серверные стойки)
- Оружие (модели для инвентаря и анимации)

**2. Тайлы и терраин (трава, камни, полы, стены) — ИИ-генерация:**

```
Gemini / Midjourney (промпт: "seamless top-down tile, [описание], 64x64")
    ↓
GIMP / Photoshop (доводка до бесшовности, цветокоррекция под палитру)
    ↓
Godot TileMapLayer
```

**Что генерируется как тайлы:**
- Терраин биомов: оранжевая трава, споровый грунт, ледяные камни, вулканическая порода
- Полы базы: металлические панели, решётки, бетон
- Стены: разные материалы и уровни прочности
- Декоративные элементы: трещины, мусор, следы, лужи

**Этапы внедрения графики:**
- Фаза 0 (прототип): Цветные квадраты — проверяем геймплей, не красоту
- Фаза 1 (вертикальный срез): Бесплатные тайлсеты-плейсхолдеры (itch.io, OpenGameArt) + первые тестовые спрайты через 3D-пайплайн
- Фаза 2–3 (основная разработка): Полная замена на финальные ассеты — тайлы через ИИ-генерацию, объекты через 3D-пайплайн
- Фаза 4 (полировка): Normal maps для всех ключевых спрайтов, финальная цветокоррекция

**Normal maps из Blender:**
При рендере спрайтов в Blender дополнительно экспортируется карта нормалей для каждого спрайта. В Godot 4 normal maps назначаются на Sprite2D и взаимодействуют с системой 2D-освещения — свет по-разному ложится на поверхность в зависимости от направления, создавая ощущение объёма.

### 15.4 Система освещения

**Технология:** Godot 4 встроенная система 2D Lights & Shadows (PointLight2D, DirectionalLight2D, LightOccluder2D).

**Почему 2D-освещение, а не 3D:**
- Полноценные динамические тени от всех объектов (стены, мебель, персонаж, фауна)
- Высокая производительность — 2D-тени это экранные эффекты, не расчёт 3D-геометрии
- Интеграция с существующим CanvasModulate (цикл дня/ночи уже работает)
- Normal maps на спрайтах дают ощущение 3D без 3D-движка

**Компоненты системы:**

**CanvasModulate** (уже есть — DaylightSystem):
- Базовое затемнение всей сцены
- Управляет циклом дня/ночи: день — полная яркость, ночь — тёмно-синий мрак
- Рассветы и закаты с плавными переходами цвета

**PointLight2D — точечные источники света:**
- Костёр, факел, лампа — тёплый оранжевый свет с мерцанием (анимация energy)
- Фонарик игрока — привязан к персонажу, движется вместе с ним
- Генераторы и оборудование — холодный голубоватый свет (техника)
- Споры в Синхронизацию — голубое свечение (эвент)
- Кондуит Предтеч — пульсирующий зеленоватый свет

**LightOccluder2D — объекты, отбрасывающие тени:**
- Стены базы (автоматически — при размещении стены добавляется occluder)
- Персонаж (маленький полигон по форме спрайта)
- Фауна (Очистители отбрасывают тени при свете фонарика — жутко)
- Мебель и оборудование внутри базы
- Деревья и крупные объекты на поверхности

**Настройки теней:**
- Фильтрация: PCF7 (мягкие тени) для атмосферных сцен, None (пиксельные) для стилизованных
- Длина теней настраивается — внутри базы короткие, от фонарика на улице — длинные
- Цвет теней — не чёрный, а тёмно-синий/фиолетовый (атмосферность)

**Атмосферные эффекты через освещение:**
- *«В базе хорошо»*: Тёплые PointLight2D на костре/лампах, мягкие тени, уютный контраст
- *«Снаружи страшно»*: Только фонарик игрока. Тени от Очистителей мелькают в темноте. Далёкие голубые огни Фильтровальщиков
- *Данжи Предтеч*: Полная темнота → включаешь генератор → зеленоватый свет, длинные тени, пробуждение защитных систем
- *Синхронизация спор (эвент)*: Споры светятся голубым — множество мелких PointLight2D по всей карте. Красиво и жутко одновременно

**Геймплейное значение освещения:**
- Фонарик потребляет энергию батареи скафандра → нужно экономить
- Очистители слепые, но видны их тени → можно заметить врага до столкновения
- Освещённая база видна издалека → ориентир для возвращения, но и маяк для фауны
- В данжах выбор: включить свет (видишь, но пробуждаешь защиту) или красться в темноте

## 16. Звуковой дизайн (концепция)

### 16.1 Философия

Звук — главный инструмент атмосферы. За стенами базы всегда слышен ветер, далёкие крики существ, скрип конструкций. Внутри — гул генераторов, шипение воздуховодов, тишина.

### 16.2 Ключевые элементы

- **Ветер** — постоянный фоновый звук, меняется по силе с погодой
- **Споры** — лёгкий, едва уловимый звон/шелест при высокой концентрации
- **Дыхание** — усиливается при гипоксии, надрывное при беге с малым O₂
- **Фауна** — далёкие крики Очистителей ночью. Тяжёлые шаги Фильтровальщиков
- **База** — гул генераторов, клацанье реле, шипение шлюза. «Звук дома»
- **Данжи** — тишина. Потом — механическое пробуждение, скрежет, вой
- **Музыка** — эмбиентная, минимальная. Нарастает в моменты опасности. Тёплая при возвращении на базу

## 17. Мультиплеер (будущая фаза)

### 17.1 Формат

Кооп на 2–4 игрока. Каждый управляет своим инженером на общей карте.

### 17.2 Архитектурные требования (закладываются сейчас)

- **Авторитарный сервер:** Один игрок — хост. Состояние мира хранится на хосте
- **Синхронизация сущностей:** Система должна поддерживать множественных игроков с первого дня на уровне архитектуры (ID-система для объектов, чёткое разделение клиент/сервер)
- **Общая база:** Все игроки строят одну базу (или несколько аванпостов)
- **Независимые вылазки:** Игроки могут разделиться (чанки загружаются вокруг каждого)
- **Система сохранений в коопе:** Лоровая система работает для каждого игрока отдельно

### 17.3 Что НЕ планируется

- PvP (только кооп)
- Выделенные серверы (только P2P через хост)
- MMO или большие сервера

## 18. Система модов (Mod API)

### 18.1 Философия

Моды — это способ дать сообществу расширять игру после релиза. Архитектура должна быть модульной с первого дня.

### 18.2 Что модеры могут менять

**Уровень 1 — Контент (простые моды):**
- Новые предметы, рецепты крафта
- Новые виды фауны (спрайты + параметры поведения)
- Новые типы построек и мебели
- Косметика: скины скафандров, текстуры стен
- Новые эвенты на основе существующей системы

**Уровень 2 — Системы (продвинутые моды):**
- Новые биомы с правилами генерации
- Новые деревья технологий / ветки дешифровки
- Новые типы ресурсов и цепочки переработки
- Новые механики скафандра (модули)
- Балансировка параметров (скорость, урон, расход ресурсов)

**Уровень 3 — Тотальные конверсии (хардкорные моды):**
- Замена всего контента (total conversion)
- Новые планеты с уникальной механикой
- Новые пути развития (помимо Терраформера и Адаптации)
- Изменение core-механик (новые шкалы выживания, новые системы)

### 18.3 Техническая реализация

**Файловая структура мода:**
```
/mods/
  /my_awesome_mod/
    mod.json          ← Метаданные (имя, версия, зависимости)
    /scripts/         ← GDScript-файлы (переопределение поведения)
    /assets/          ← Спрайты, звуки, тайлсеты
    /data/            ← JSON/TRES: рецепты, предметы, существа, биомы
    /localization/    ← Переводы
```

**Принципы архитектуры для поддержки модов:**
- Все данные (предметы, рецепты, существа, биомы) хранятся в JSON/Resource-файлах, НЕ хардкодятся
- Система регистрации: каждый предмет/существо/рецепт имеет уникальный ID
- Event Bus: события игры (крафт, смерть, постройка) публикуются в шину, моды подписываются
- Хуки: ключевые системы имеют точки расширения (before_craft, after_build, on_damage и т.д.)
- Ресурс-лоадер: загрузка ассетов через абстракцию, которая ищет сначала в модах, потом в базовой игре

### 18.4 Реестр модов

- Steam Workshop (основная платформа распространения — при релизе в Steam)
- Ручная установка (папка /mods/)
- Система порядка загрузки и разрешения конфликтов

```
│   └── particles/
├── scenes/                  ← Godot-сцены
│   ├── world/
│   ├── ui/
│   ├── player/
│   └── buildings/
├── mods/                    ← Папка для модов (загружается ModLoader)
└── saves/                   ← Сохранения игр
```

### 19.3 Ключевые паттерны

**Event Bus (Шина событий):**
```
Любое действие в игре → EventBus.emit("action_name", data)
Системы и моды подписываются → EventBus.connect("action_name", handler)
```
Примеры событий: `item_crafted`, `building_placed`, `creature_killed`, `filter_clogged`, `spore_breach`, `technology_decrypted`

**Registry (Реестр):**
Все игровые объекты регистрируются по уникальному ID:
```
ItemRegistry.register("base:iron_ingot", iron_ingot_data)
FaunaRegistry.register("base:cleaner_alpha", cleaner_data)
```
Моды добавляют свои записи с префиксом мода: `"my_mod:super_sword"`

**Data-Driven Design:**
Максимум параметров — в файлах данных, минимум хардкода:
```json
{
  "id": "base:iron_ingot",
  "name": "Железный слиток",
  "weight": 2.5,
  "stack_size": 20,
  "smelting_time": 5.0,
  "source_ore": "base:iron_ore"
}
```

### 19.4 Система сохранений

**Структура файла сохранения:**
```
/saves/save_001/
├── meta.json         ← Информация о сейве (дата, версия, мод-лист)
├── player.json       ← Статы, инвентарь, позиция, мутации
├── base.json         ← Постройки, коммуникации, состояние систем
├── world_seed.json   ← Seed мира + параметры генерации
├── chunks/           ← Изменения в чанках (только модифицированные)
│   ├── chunk_0_0.bin
│   ├── chunk_12_-5.bin
│   └── ...
├── tech_tree.json    ← Прогресс дешифровки
├── events.json       ← Состояние системы событий
└── lore_progress.json ← Какие записи найдены
```

## Приложение A — Глоссарий

| Термин | Значение |
|--------|----------|
| **Ковчег** | Орбитальный корабль-город, дом человечества после гибели Земли |
| **Инженер** | Специально подготовленный разведчик-колонист, посланный с Ковчега |
| **Предтечи** | Древняя цивилизация (на самом деле — предки людей), жившая на планете |
| **Очистители** | Агрессивная фауна, реагирующая на шум |
| **Фильтровальщики** | Мирные существа, очищающие воздух от спор |
| **Голиаф** | Гигантская биомеханическая машина Предтеч |
| **Дешифровка** | Процесс восстановления технологий из повреждённого блока данных |
| **Синхронизация спор** | Событие: споры светятся, фауна агрессивна |
| **Шлюз** | Камера между внутренним и внешним пространством базы |
| **Изначальный** | Финальная форма трансформации на пути Адаптации |
| **Споровый ферментатор** | Источник энергии средней игры: споры при ферментации выделяют электричество |
| **Сейсмоуловитель** | Пьезоэлектрический генератор, преобразующий вибрации планеты в энергию |
| **Кондуит Предтеч** | Рабочий энергоузел древней цивилизации, к которому можно подключиться |
| **Корень-сердце** | Живой организм-симбионт (путь Адаптации), вросший в базу и дающий биоэлектричество |
| **Плазменный реактор** | Венец инженерии (путь Терраформера), удержание плазмы в магнитной ловушке |
| **Normal map** | Карта нормалей — текстура, указывающая направление поверхности для реалистичного 2D-освещения |
| **Pre-rendered** | Метод создания 2D-спрайтов из 3D-моделей (как в Factorio) |
