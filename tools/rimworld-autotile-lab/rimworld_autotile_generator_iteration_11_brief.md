# Iteration 11 - PBR Pipeline

## Goal

Extend the generator export pipeline so the current authored result can produce:

- export-only `height`, `ORM`, `emission`, and `flow` atlases
- Godot-ready `TerrainShapeSet` and `TerrainMaterialSet` `.tres` files
- atlas bleed padding suitable for runtime sampling
- seam lint output and deterministic build proof for the current recipe

This iteration remains editor-only and tool-side. It must not change runtime
packet contracts, commands, events, or public APIs in the game code.

## Runtime classification

- runtime work class: `interactive`
- authoritative source of truth: editor `state`, current params, material maps,
  and generated export canvases inside
  `rimworld_autotile_generator_runtime_export.js`
- derived state: padded atlases, `.tres` text exports, seam-lint metrics,
  deterministic proof hashes, ZIP bundle contents
- dirty unit: one export rebuild triggered by material / color / shape /
  variant changes
- single write owner: generator export pipeline in
  `rimworld_autotile_generator_runtime_export.js`

## Boundary contract check

- Existing safe path to reuse: current tool-side JSON recipe and exported PNG
  bundle path.
- `system_api.md`, `event_contracts.md`, `packet_schemas.md`, and
  `commands.md` were checked before code.
- This iteration must stay compatible with the existing canonical
  `TerrainShapeSet` / `TerrainMaterialSet` resource fields already defined in
  `data/terrain/*.gd`.
- If implementation drifts into new runtime resource fields or runtime-facing
  API contracts, stop and update canonical docs in the same task.

## Non-Goals

- no new runtime shader family or gameplay-side registry wiring
- no canonical `TerrainMaterialSet` field expansion in `docs/`
- no new map-editing tools or shape-authoring controls
- no broad file split or architecture refactor of the generator

## Allowed Files

- `tools/rimworld-autotile-lab/rimworld_autotile_generator.html`
- `tools/rimworld-autotile-lab/rimworld_autotile_generator_runtime_export.js`
- `tools/rimworld-autotile-lab/rimworld_autotile_generator_active_epic.md`
- `tools/rimworld-autotile-lab/rimworld_autotile_generator_iteration_11_brief.md`

## Forbidden Files

- files outside `tools/rimworld-autotile-lab/`
- canonical docs in `docs/` unless runtime/export contract drift is proven
- runtime GDScript systems under `core/` or `data/terrain/`

## Implementation Steps

1. Add export-only channel generation for:
   - `height atlas`
   - `ORM atlas`
   - `emission atlas`
   - `flow atlas`
   - `top normal`
   - `face normal`
2. Add bleed padding to atlas exports without changing the core 47-case catalog.
3. Add seam-lint metrics and deterministic build proof to the generated export
   payload.
4. Add Godot `.tres` text export for:
   - `TerrainShapeSet`
   - `TerrainMaterialSet`
5. Keep recipe export/import, ZIP bundle, and current visible preview pipeline
   compatible.

## Risks

- export-only channels can drift from current material intent if derived too
  naively from layer data
- padded atlas layout can invalidate manifest assumptions if geometry metadata
  is not updated together
- deterministic proof can become misleading if it hashes only filenames instead
  of actual generated bytes

## Smoke Tests

- `node --check tools/rimworld-autotile-lab/rimworld_autotile_generator_runtime_export.js`
- `git diff --check -- tools/rimworld-autotile-lab/rimworld_autotile_generator_runtime_export.js tools/rimworld-autotile-lab/rimworld_autotile_generator.html tools/rimworld-autotile-lab/rimworld_autotile_generator_active_epic.md tools/rimworld-autotile-lab/rimworld_autotile_generator_iteration_11_brief.md`
- grep for `heightAtlas`, `ormAtlas`, `emissionAtlas`, `flowAtlas`,
  `buildShapeSetTres`, `buildMaterialSetTres`, `seam lint`, and
  `deterministic proof`

## Definition of Done

- the generator can download the new PBR-oriented export atlases
- the ZIP bundle includes those atlases plus Godot `.tres` files
- export metadata records padded atlas geometry, seam-lint results, and
  deterministic proof
- no canonical doc updates are required, or grep proof is recorded if they are
