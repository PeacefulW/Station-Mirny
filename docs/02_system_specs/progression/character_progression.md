# Character Progression

## Purpose

Character progression should make the engineer feel increasingly competent without discarding the survival identity of the game.

## Scope

This spec owns:
- stats
- use-based skills
- suit module progression
- inventory carrying model

## Stats

High-level intended stats include:
- strength
- endurance
- intelligence
- spore resistance
- oxygen efficiency
- thermoregulation

## Skill model

The intended rule is use-based growth:
- build to improve engineering
- carry and labor to improve physical capacity
- decrypt and analyze to improve intellect
- endure harsh exposure to improve specialized survival traits

## Equipment progression

### Suit

The suit is modular, with meaningful slots such as:
- filter
- oxygen tank
- battery
- armor
- utility

### Weapons

Weapon progression should move through:
- inherited emergency gear
- primitive survival tools
- crafted firearms
- branch-specific late gear

## Inventory

Inventory is slot-based with weight pressure.

Desired effect:
- expeditions are shaped by preparation and carrying choices
- base storage remains important

## Acceptance criteria

- progression changes capability and planning, not just raw numbers
- the suit feels like a core identity object
- use-based growth rewards actual playstyle

## Failure signs

- stats exist but do not affect choices
- suit modules are obvious linear upgrades only
