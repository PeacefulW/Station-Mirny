# Sprint 01 Rules (DoR / DoD)

## Definition of Ready (DoR)
Задача может стартовать, если:
1. Есть ясный результат (что должно измениться для игрока/системы).
2. Понятны файлы/модули, где будет работа.
3. Описан способ проверки (авто/ручной сценарий).
4. Указаны зависимости и риск.
5. Оценка времени дана в часах.

## Definition of Done (Task-level)
Задача считается закрытой, если:
1. Реализация завершена, код читаемый и типизированный.
2. Нет нарушений базовых архитектурных правил проекта (data-driven, EventBus/Registry и т.п.).
3. Обновлены нужные данные/документация.
4. Пройдены релевантные smoke-проверки.
5. Нет P0/P1 багов, появившихся из-за задачи.

## Definition of Done (Sprint-level)
Спринт считается успешным, если:
1. Фаза 0 закрыта фактически (включая простой крафт).
2. Core-loop (старт/сбор/стройка/бой/save-load) стабилен.
3. Документация отражает реальное состояние репозитория.
